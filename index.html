ID Reader

Purpose

The purpose of the ID Reader project is to create a more secure, agile and GDPR-compliant way to register people entering and exiting a sensitive room. The solution should also meet requirements that can be set according to ISO standards and serve as a proof of concept for how simple technology like a Raspberry Pi can be used in a security and logging context.

Problem

Currently, a manual system is used where people entering and exiting a specific room must write down their name and time of entry and exit on a physical piece of paper. This poses several problems:

  • Time-consuming: It takes time to manually fill in the information each time.
  • Lack of security: The information is easily accessible to unauthorized people and can be manipulated or stolen.
  • Not GDPR-safe: Personal data is stored openly and can be read by others.
  • Limited traceability and no automation: It is difficult to perform analysis or control access afterwards.
  • Not ISO compliant: The current system does not meet the requirements for security and traceability.

Solution

The solution is a digital system where the user places their ID card in a drawer. A camera connected to a Raspberry Pi takes a picture of the card. When the user presses a button, a Python program is activated that reads the name using image processing/OCR. The information is sent via MQTT to another Raspberry Pi that stores the name along with a timestamp in a SQL database.

  • Automates the process and saves time.
  • Protects information by storing it digitally instead of physically.
  • Enables traceability with precise logging of entry and exit.
  • Can be expanded for additional security features (e.g. door control, authentication).
  • Better complies with GDPR by minimizing manual handling of personal data.
  • Can be adapted to meet ISO requirements for security and traceability.

Modern Data Stack Implementation for Job Ads Analytics

Purpose

The project aims to implement the modern data stack to solve a real-world problem. In addition to using a certain number of technologies, there is a focus on working together in a data team. This includes both agile development and being able to use git and github in a team.

Problem

The company Arbetskollen AB wants to build a data pipeline that serves a dashboard to visualize statistics for job advertisements. This could be a dimension to help future students find the right field to study in or for job seekers to more easily navigate between different professions.

Solution

The final solution is a robust data pipeline and reporting system built to ingest, transform, and visualize job advertisement data from Arbetsförmedlingens API. The data modeling is based on a star schema, chosen for its simplicity and efficiency in supporting analytical queries and reporting. Data flows through a well-structured data architecture with three layers:

  • Staging layer (schema: staging), Stores raw API data.
  • Transformation layer (schema: warehouse), Cleans and processes data using DBT.
  • resentation layer (schema: marts), Contains aggregated and refined data ready for reporting and visualization.

The pipeline uses Python and DLT to fetch and load data into Snowflake, while DBT performs advanced transformations and metric calculations. The final data is visualized using a Streamlit dashboard, which includes a home screen and navigation menu for improved user experience. This architecture ensures clean, reliable, and easily accessible data for stakeholders, with efficient performance and maintainability.

Sales data

Purpose

The purpose of this Power BI dashboard is to empower decision-makers with real-time, data-driven insights into company sales performance by integrating and visualizing sales data from a centralized SQL database.

Problem

The company has a large amount of sales data stored in CSV files that need to be processed and loaded into a SQL database for analysis and reporting. The current manual process is time-consuming and error-prone, leading to delays in data availability and potential inaccuracies in reporting.

Solution

To address the inefficiencies and risks associated with manually processing large volumes of sales data from CSV files, I implemented an automated data pipeline that extracts, transforms, and loads (ETL) the data into a SQL database using Python. Once the data is available in SQL, it is connected to Power BI for real-time dashboards and interactive reporting. Power BI will allow stakeholders to:

  • Visualize sales trends and patterns over time.
  • Analyze sales performance by product, region, and sales representative.
  • Identify top-performing products and areas for improvement.
  • Make data-driven decisions to optimize sales strategies and increase revenue.

Data Pipeline & AI/ML

Purpose

The purpose of the project is to modernize DataVision Analytics AB's data platform to enable the use of artificial intelligence (AI) and machine learning (ML). The goal is to analyze large amounts of data from customer interactions, transactions and market research. Which leads to more advanced data analysis that will help customer understanding, optimized sales and better customer experience.

Problem

DataVision Analytics AB wants to update its data system so that it can think smarter using AI and Machine Learning. They want to learn more about what customers like and how they shop, so they can sell more and make their customers happier. The company has tons of information about its customers from purchases, conversations, and surveys. They plan to use a powerful data system called Microsoft Azure to make this possible.

Solution

Using Azure Data Factory, the company will collect and organize data from:

  • Customer interactions (e.g., website visits, emails, support chats)
  • Transactions (in-store and online purchases)
  • arket research (surveys, social media, external reports)

Data Storage:
  • Data Lake, stores all types of data, even unstructured (e.g., text, images, audio).
  • Data Warehouse, a structured database used for fast and reliable data analysis.

The company will use ETL (Extract, Transform, Load) methods to:
  • Combine and clean data from different systems.
  • Ensure consistency and accuracy.

Using Azure Machine Learning Studio, the company will:
  • Train ML models with historical data (customer behavior, purchases, etc.)
  • Predict future events like Which products will sell well and which customers may stop buying.
  • Continuously update models with new data to keep them accurate

AI models will support:
  • Predictive analytics: Forecasting sales and customer churn
  • Personalization: Tailored offers and product recommendations
  • Automated analysis: Faster decisions with minimal manual effort
