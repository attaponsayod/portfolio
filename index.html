My Work

ID Reader

Purpose

The purpose of the ID Reader project is to create a more secure, agile and GDPR-compliant way to register people entering and exiting a sensitive room. The solution should also meet requirements that can be set according to ISO standards and serve as a proof of concept for how simple technology like a Raspberry Pi can be used in a security and logging context.

Problem

Currently, a manual system is used where people entering and exiting a specific room must write down their name and time of entry and exit on a physical piece of paper. This poses several problems:

  • Time-consuming: It takes time to manually fill in the information each time.
  • Lack of security: The information is easily accessible to unauthorized people and can be manipulated or stolen.
  • Not GDPR-safe: Personal data is stored openly and can be read by others.
  • Limited traceability and no automation: It is difficult to perform analysis or control access afterwards.
  • Not ISO compliant: The current system does not meet the requirements for security and traceability.

    • Solution

    The solution is a digital system where the user places their ID card in a drawer. A camera connected to a Raspberry Pi takes a picture of the card. When the user presses a button, a Python program is activated that reads the name using image processing/OCR. The information is sent via MQTT to another Raspberry Pi that stores the name along with a timestamp in a SQL database.

  • Automates the process and saves time.
  • Protects information by storing it digitally instead of physically.
  • Enables traceability with precise logging of entry and exit.
  • Can be expanded for additional security features (e.g. door control, authentication).
  • Better complies with GDPR by minimizing manual handling of personal data.
  • Can be adapted to meet ISO requirements for security and traceability.
